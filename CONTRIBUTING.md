# Contributing to DCL

Thank you for your interest in contributing to the Declarative Capability Language (DCL).

## Ways to Contribute

Contributions are welcome in many forms:

* Bug reports
* Compiler improvements
* Documentation improvements
* Language examples
* Playground development
* Tooling and editor support
* Language design discussions

## Before Making Language Changes

DCL is a language project with a semantics-first design philosophy.

Language changes should be discussed before implementation.

For significant language changes:

1. Open an issue describing the proposal.
2. Explain the motivation.
3. Describe semantic implications.
4. Discuss compiler and runtime implications.
5. Await feedback before implementation.

## Pull Requests

Please:

1. Fork the repository.
2. Create a feature branch.
3. Keep changes focused and well scoped.
4. Include tests where appropriate.
5. Ensure the compiler continues to pass all tests.
6. Submit a pull request with a clear description.

## Documentation

Documentation improvements are always welcome.

Examples, tutorials, and clarifications are valuable contributions.

## Code of Conduct

Be respectful, constructive, and collaborative.

Healthy technical disagreement is encouraged.
Personal attacks are not.

## Questions

If you are unsure whether a contribution aligns with the direction of the project, please open a discussion before investing significant effort.
