# DCL Governance

## Project Founder

Declarative Capability Language (DCL) was created by Russell East.

## Project Direction

The long-term direction of DCL is guided by:

* The DCL Manifesto
* Decision Records
* Versioned Language Specifications

These documents form the authoritative source of language intent and design principles.

## Maintainers

Project maintainers are responsible for:

* Reviewing contributions
* Managing releases
* Maintaining specification consistency
* Preserving language quality

## Language Evolution

DCL evolves through discussion, experimentation, implementation, and documented decisions.

Major language changes should be supported by:

* clear motivation
* semantic justification
* compiler implications
* example usage

## Contributions

Contributions are welcome and encouraged.

Acceptance of contributions remains the responsibility of the project maintainers.

## Open Governance

The project aims to be transparent in its decision making.

Design discussions, issues, and proposals should be conducted openly whenever possible.
